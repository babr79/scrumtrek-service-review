# Data-Driven Organization

← [AI-направление](обзор-ai.md) · [Карта уровней](каталог-услуг.md)

> Консалтинг по построению data-driven культуры: от хаотичного использования данных к системному управлению дата-продуктами.

## На каком уровне работает

**Стратегический + координационный.** Data-driven организация — это стратегическое решение (уровень CDO/CTO), но реализуется через координацию: федерализированные дата-команды, Data Governance Council, стандарты между подразделениями.

## Для кого

- CDO / CTO / CPO, которым нужна методология работы с данными
- Команды аналитики и Data Science без структуры и процессов
- Руководители, видящие разрыв между инвестициями в данные и бизнес-результатами

## Проблема

Типичные симптомы:
- Данные разрозненны по подразделениям (Data Silos)
- Нет ответственных за качество и доступность
- Data Science как сервис без ответственности за бизнес-результат
- Пожары обнаруживаются после их начала
- Чувствительные данные доступны тем, кому не следует

## Модель зрелости данных

| Уровень | Название | Описание |
|---|---|---|
| 0 | Data Free | Решения на опыте и «чуйке» |
| 1 | Data Aware | Долгосрочные решения: KPI, квоты, планы |
| 2 | Data Silos | Локальное использование внутри подразделений |
| 3 | Data Integrated | Данные — важный актив. Используются по всей организации |
| 4 | Optimizing | Оптимизация инфраструктуры. Децентрализация |

## Пять блоков консалтинга

### Блок 1: Аудит и стратегия данных
- Оценка текущего уровня зрелости
- Идентификация болезненных участков
- Data Related Goals, Data Impact Map

### Блок 2: Data Products
- Определение дата-доменов и дата-продуктов
- Типология: сырые данные → подготовленные → алгоритмы (ML) → поддержка решений → автоматизация
- Свойства: Discoverable, Addressable, Self-describing, Trustworthy & Secure
- SLA/SLO для дата-продуктов

### Блок 3: Data Governance
- Data Policies — регламент правил
- Каталог дата-продуктов и ответственных
- Бизнес-глоссарий — общие термины и методики расчёта
- Data Governance Council

### Блок 4: Структура команд
- Федерализированная модель: независимые дата-команды по доменам
- Роль Data Product Owner
- Платформенная команда — Self-service инструменты
- CDO за методологию и платформу, CPO — за бизнес-составляющую

### Блок 5: DataOps / MLOps
- Автоматизация пайплайнов данных
- Мониторинг качества данных
- CI/CD для дата-продуктов
- Проактивный контроль качества

## Ключевые концепции

| Концепция | Суть |
|---|---|
| Data Product | Продукт, использующий данные для бизнес-цели |
| Data Mesh | Децентрализованная архитектура: домен-ориентированные команды |
| Data Governance | Методология управления данными |
| DataOps / MLOps | Управление пайплайнами данных и ML-моделями |
| Data Platform | Self-service инструменты для дата-команд |

## Форматы

| Формат | Описание |
|---|---|
| Workshop | Аудит зрелости + roadmap (1–2 дня) |
| Консалтинг | Проектирование структуры, governance, платформы (2–4 мес.) |
| Сопровождение | Менторская поддержка при внедрении (6–12 мес.) |

## Результат

- Оценка зрелости и целевое состояние
- Каталог дата-продуктов с владельцами и SLA
- Структура дата-команд
- Data Governance framework
- Roadmap перехода
