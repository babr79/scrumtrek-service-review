# AI для разработчиков

← [AI-направление](../обзор-ai.md) · [Карта уровней](../../../каталог-услуг.md)

> Интеграция AI-инструментов в процесс разработки ПО — от IDE до продакшена.

## На каком уровне работает

**Тактический уровень.** Работает непосредственно с инженерными командами — учит интегрировать AI в каждый этап SDLC.

На координационном уровне может косвенно помогать через стандартизацию AI-инструментов и подходов между командами, но основная ценность — тактическая.

## Для кого

- Разработчики среднего и старшего уровня (стек-агностичная программа)
- Техлиды и архитекторы
- DevOps-инженеры и QA

## Проблема

AI кардинально меняет процесс разработки ПО. Но большинство разработчиков используют AI на уровне автокомплита в IDE. Потенциал AI в SDLC — от формулировки требований до деплоя — остаётся нераскрытым.

## Программа: 8 сессий × 4 часа (32 часа)

| Сессия | Тема | Содержание |
|---|---|---|
| 1 | AI-инструменты в IDE | GitHub Copilot, Claude Code, Cursor. Продвинутые техники: контекст, инструкции, шаблоны. Метрики эффективности |
| 2 | AI в SDLC | Как AI меняет каждый этап: требования, проектирование, код-ревью, тестирование, документация. Новые процессы и роли |
| 3 | GenAI и LLM изнутри | Архитектура LLM, токенизация, инференс. Выбор модели под задачу. Локальные vs облачные модели. Безопасность |
| 4 | API и SDK интеграция | Подключение LLM через API. Streaming, function calling, structured output. Обработка ошибок и fallback |
| 5 | RAG и работа с данными | Эмбеддинги, векторные БД, пайплайны. Построение RAG для корпоративной базы знаний. Оценка качества |
| 6 | MCP и AI-агенты | Model Context Protocol. Мультиагентные системы. Оркестрация агентов. Практика: создание AI-агента |
| 7 | Мультимодальность и локальные модели | Vision, аудио, видео. Запуск моделей локально. Оптимизация и квантизация. Fine-tuning |
| 8 | Capstone: деплой AI-продукта | Финальный проект: от идеи до деплоя. CI/CD для AI. Мониторинг и масштабирование |

## Инструменты

| Категория | Инструменты |
|---|---|
| AI в IDE | GitHub Copilot, Claude Code, Cursor, Windsurf |
| LLM API | OpenAI API, Anthropic API, российские LLM |
| RAG | LangChain, LlamaIndex, векторные БД |
| Агенты | n8n, Claude Code, собственные реализации |
| Инфраструктура | Docker, Supabase, Vercel, Railway |

## Форматы проведения

| Формат | Описание |
|---|---|
| Корпоративный | Адаптация под стек и процессы компании. Группа до 20 человек |
| Открытый | Сборная группа, стек-агностичная программа |

## Длительность

32 часа (8 сессий × 4 часа). Обычно проводится в течение 4 недель.

## Результат

- Разработчики интегрируют AI на каждом этапе SDLC
- Умеют строить RAG-пайплайны и AI-агентов
- Понимают архитектуру LLM и могут выбрать подходящее решение
- Каждый завершает курс с работающим AI-проектом
