# Обучение проведению Event Storming онлайн

← [Назад к направлению](обзор-инженерии.md) · [Все услуги](каталог-услуг.md)

> Event Storming — гибкий формат воркшопа для совместного исследования сложных областей бизнеса. Этот курс учит самостоятельно проводить сессии Event Storming: от подготовки до интеграции результатов в рабочие процессы.

## На каком уровне работает

| Уровень | Применимость | Как именно работает на этом уровне |
|---|---|---|
| Стратегический | ✔ | Дизайн новых бизнес-линий, исследование жизнеспособности бизнес-моделей стартапов |
| Координационный | ✔ | Фасилитация межкомандных сессий, выявление бизнес-требований, выстраивание общего понимания |
| Тактический | ✔ | Software Design Modelling, работа с агрегатами, проектирование микросервисных решений |

## Программа курса

### Введение в Event Storming
Обзор метода, цели и задачи. Теория сложных адаптивных систем — почему классические подходы к анализу не работают в сложных доменах.

### Совместное исследование событий
Выявление и анализ ключевых бизнес-событий. Как вытащить знания из голов экспертов предметной области.

### Подготовка к сессии
Особенности онлайн-формата. Формирование команды участников. Использование AI для подготовки и анализа.

### Фокус исследования и поток событий
Определение фокуса сессии. Хронологизация событий — выстраивание временной шкалы бизнес-процесса.

### Big Picture исследование
Анализ системы целиком. Выделение структур, акторов, доменов. Поиск «горячих точек» и противоречий.

### Моделирование процессов
Работа с событиями на уровне отдельного процесса. Каскадные события. Развитие нотации Event Storming.

### Software Design Modelling
Переход от бизнес-модели к архитектуре ПО. Исследование агрегатов с помощью AI. Выделение bounded contexts.

### Интеграция в рабочие процессы
Как встроить Event Storming в регулярные процессы команды. Практические рекомендации по частоте и формату проведения.

## Результаты обучения

Участники смогут самостоятельно:
- Проводить сессии Event Storming в онлайн-формате
- Выявлять, анализировать и структурировать бизнес-события
- Выделять агрегаты и ограниченные контексты
- Интегрировать результаты в существующие процессы разработки
- Применять методы визуализации и фасилитации
