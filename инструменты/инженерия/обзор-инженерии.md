## Как услуги связаны между собой

```
                    Диагностика IT-устойчивости
                     (анализ текущего состояния)
                              │
                    ┌─────────┼─────────┐
                    ▼         ▼         ▼
              Event Storming  DDD    Школа Архитектора
              (исследование)  (модель)  (компетенции)
                    │         │         │
                    └─────┬───┘         │
                          ▼             │
                   Проектирование       │
                   микросервисов ◄──────┘
                          │
                          ▼
                   AI-контроль
                   качества кода
```

Типичный путь клиента: диагностика выявляет проблемы → Event Storming исследует предметную область → DDD задаёт структуру → микросервисы реализуют архитектуру → AI помогает контролировать качество. Школа Архитектора даёт теоретическую базу для всего пути.
