# Основы Канбан-систем (KSD / Base)

← [Назад к направлению](../обзор-agile.md) · [Все услуги](../../../каталог-услуг.md)

> Тренинг для проектных менеджеров, линейных менеджеров, руководителей подразделений и тимлидов, которым важно обеспечивать предсказуемость сроков и управлять производительностью своего подразделения на основе метрик. Научимся оцифровывать работу подразделения с помощью процессных метрик (Lead Time, Throughput, WIP и других), и прогнозировать время выполнения с вероятностью попадания 85-90%

## На каком уровне работает

| Уровень | Применимость | Как именно работает на этом уровне |
|---|---|---|
| **Стратегический** | ✘ | Курс не затрагивает стратегический уровень |
| **Координационный** | ✔ | Каденция по пополнению, квоты на конкретного заказчика, классы обслуживания, Service Request Manager |
| **Тактический** | ✔ | Процессные метрики работа команд, прогнозирование Lead Time, управления WIP-лимитами, управление потоком задач |

## Для кого

- Руководители, которые хотят прогнозировать сроки на основе данных, а не интуиции
- PM/PO, которым нужно управлять сроками и потоком задач
- Тимлиды и линейные менеджеры

## Ключевая ценность

**«Как обеспечить предсказуемость сроков и эффективность, не увеличивая ресурсы»** — практический подход к управлению потоком с помощью методологии Канбан-метод.

Для руководителей подразделений: «Прогноз сроков бизнес-задачи с точностью 85-90%».
Для бизнеса/заказчиков: оптимизация процессов без сопротивления и увольнений, на основе статистических данных.
Для сотрудников: «Работа без авралов и переработок» с загрузкой на основе реальных возможностей.

## После тренинга вы сможете:
- Давать прогноз сроков выполнения задач с высокой достоверностью (85–90%) на основе Lead Time
- Уверенно отвечать на вопросы бизнес-заказчика о сроках и времени взятия в работу, на основе статистических данных
- Управлять производительностью рабочего процесса через WIP limits и статистику Throughput
- Проектировать Канбан-доску для эффективного управления подразделением и сбора метрик 
- Разработать правила и политики управления работой своего подразделения, для снижения нагрузки на менеджмент и принятие решений
- Органически Встроить в работу подразделения управленческие мероприятия для эффективной работы с потоком задач и регулярного анализа эффективности потока
- Управлять входящим потоком задач на основе класса сервиса и метрик
- Оценить экономику "вбросов" срочных задач, для прозрачной аргументации в разговоре с бизнес-заказчиком

## Программа

- Визуальный менеджмент потоком задач
- Метрики рабочего процесса: Lead Time, Throughput, Work In Progress (WIP), WIP Age и другие 
- Прогнозирование времени выполнения задач с вероятностью 85-90% на основе метрик рабочего потока
- WIP-лимиты: как управлять производительностью подразделения меняя этот параметр
- Экономика "вбросов" срочных задач, и как управлять потерями
