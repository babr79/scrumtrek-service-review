# Координационный уровень

← [Каталог услуг](каталог-услуг.md)

> Когда стратегия понятна, но страдает исполнение из-за рассинхрона подразделений — помогаем выстроить операционный ритм. Этот уровень про связность: синхронизация десятков команд, управление зависимостями, масштабирование практик, единые стандарты. Все четыре инструмента ScrumTrek работают здесь — обеспечивая горизонтальную координацию через свою оптику.

---

## Кто клиент на этом уровне

- Руководители направлений, директора программ
- Трайб-лиды, RTE (Release Train Engineer)
- Руководители PMO, CDO, Head of Engineering
- AI Champions и владельцы центров компетенций
- Руководители, управляющие 5+ продуктовыми командами

## Какие задачи решаем

- Команды работают, но не синхронизированы — зависимости блокируют поставку
- Масштабируем Agile с одной команды на десятки — нужен фреймворк
- Стратегия каскадирована в OKR, но департаменты тянут в разные стороны
- Нужен AI-центр компетенций, чтобы AI-инициативы не застревали в пилотах
- Архитектура команд не соответствует архитектуре продукта (проблема Conway's Law)
- Нужно единое информационное пространство для принятия решений

## Стратегатор на этом уровне

На координационном уровне Стратегатор обеспечивает операционное управление портфелем: поток инициатив между стратегией и командами, синхронизация программ, каскадирование целей.

| Услуга | Что делает на координационном уровне | Подробнее |
|---|---|---|
| **Сборка OKR** | Каскадирование: корпоративные OKR → OKR департаментов. Синхронизация целей между функциями, разрешение конфликтов приоритетов | [→](сборка-okr.md) |
| **СтратКанбан** | Управление потоком инициатив между портфелем и командами: визуализация, WIP-лимиты, еженедельные синхронизации | [→](страт-канбан.md) |
| **Идентификация портфеля** | Классификация инициатив по потокам, выявление дублирования между подразделениями | [→](идентификация-портфеля.md) |
| **Скоринг и триаж** | Управление потоком входящих инициатив, приоритизация между программами, WSJF | [→](скоринг-и-триаж.md) |
| **Создание ПМО** | Операционная работа ПМО: управление зависимостями между инициативами, портфельный дашборд, регулярные ревью | [→](создание-пмо.md) |
| **Экосистема инноваций** | Управление когортами в акселераторе, синхронизация инновационных команд, evidence-based приоритизация | [→](экосистема-инноваций.md) |
| **Акселератор** | Координация до 4 команд по 5 человек: еженедельные стендапы, экспертный коучинг, синхронизация результатов | [→](акселератор.md) |
| **Тренинг LPM** | Управление потоком инициатив, синхронизация программ, метрики портфеля | [→](тренинг-lpm.md) |
| **Тренинг OKR** | Каскадирование OKR между функциями, отслеживание зависимостей | [→](тренинг-okr.md) |
| **Тренинг для PMO** | Новая роль ПМО: от контроля через gate review к управлению потоком. Метрики, Agile-governance | [→](тренинг-pmo.md) |

## Agile-практики на этом уровне

На координационном уровне Agile — это масштабирование: фреймворки для синхронизации десятков команд, PI-планирование, управление зависимостями, выстраивание трайбов и ART (Agile Release Train).

### Трансформация

| Услуга | Что делает на координационном уровне | Подробнее |
|---|---|---|
| **Гемба-аудит** | Исследует Value Stream Map: как ценность проходит через несколько подразделений, где зависимости и блокеры | [→](гемба-аудит.md) |
| **Поддержка Agile-трансформации** | Обучение и запуск Agile-фреймворков на уровне нескольких команд, синхронизация между ними | [→](поддержка-трансформации.md) |
| **Консультирование ТРО** | Методологическая поддержка трансформационного офиса: координация изменений между подразделениями | [→](консультирование-тро.md) |
| **Фасилитация** | Проведение PI-планирования и межкомандных ретроспектив для десятков команд (опыт: 450+ участников) | [→](фасилитация.md) |

## AI / Кактус.AI на этом уровне

На координационном уровне AI — это связность и масштабирование: создание центра компетенций, обучение AI Champions как мостов между подразделениями, единая платформа и стандарты.

| Услуга | Что делает на координационном уровне | Подробнее |
|---|---|---|
| **AI-центр компетенций** | Основная услуга этого уровня. 6–12 месяцев на создание постоянной структуры: единые стандарты, playbook, governance, Champions как мосты между отделами | [→](ai-центр-компетенций.md) |
| **AI Champions** | Обучение внутренних AI-лидеров (9 модулей × 4ч): связывают подразделения, устанавливают единые стандарты, координируют AI-инициативы | [→](ai-champions.md) |
| **AI-стратегия и внедрение (фазы 2, 4)** | IT-инфраструктура: выбор LLM, безопасность, интеграция платформы. Масштабирование пилотов на организацию | [→](ai-стратегия.md) |
| **Data-Driven Organization** | Федерация data-команд по доменам, Data Governance Council, кросс-функциональные стандарты | [→](data-driven.md) |
| **Корпоративная ИИ-платформа** | Единая точка доступа для всех подразделений: RBAC, групповые политики, общая база знаний (RAG), межотдельные стандарты | [→](платформа.md) |

**Пример:** Фарма-компания создаёт AI-центр с 5 основателями, обучает 40 AI Champions из разных отделов. Через 6 месяцев 12 инициатив работают на единой платформе с контролем данных, 3 уже в продакшене.

## Инженерная культура на этом уровне

На координационном уровне инженерия — это синхронизация понимания между командами, границы ответственности, архитектурная документация для разных аудиторий.

| Услуга | Что делает на координационном уровне | Подробнее |
|---|---|---|
| **Диагностика IT-устойчивости** | Анализ процессов и структуры, выявление системных ограничений, рекомендации для руководителей направлений | [→](it-диагностика.md) |
| **Event Storming** | Синхронизация понимания между командами, выявление разрывов в коммуникации и системных ограничений | [→](event-storming.md) |
| **Школа Архитектора ПО** | Документирование архитектуры для разных аудиторий, оценка архитектуры (ATAM) | [→](школа-архитектора.md) |
| **Стратегические паттерны DDD** | Карта контекстов между командами, паттерны коммуникации, границы ответственности, дизайн оргструктуры по Conway's Law | [→](стратегические-паттерны-ddd.md) |
| **Обучение Event Storming** | Фасилитация межкомандных сессий, навык самостоятельного проведения | [→](обучение-event-storming.md) |
| **Проектирование микросервисов** | Организационные практики, структура команд вокруг сервисов | [→](проектирование-микросервисов.md) |

## Типичные сценарии входа

**«5+ команд, хаос в зависимостях»** → Гемба-аудит → Фасилитация PI-планирования → SAFe SM / LeSS Basics

**«OKR каскадированы, но департаменты тянут в разные стороны»** → OKR Master → СтратКанбан

**«AI-пилоты не масштабируются»** → AI-центр компетенций → AI Champions → ИИ-платформа

**«Команды не понимают друг друга»** → Event Storming → Стратегические паттерны DDD → Проектирование микросервисов

**«Нужен постоянный контроль портфеля»** → Создание ПМО → Тренинг PMO → Скоринг и триаж
